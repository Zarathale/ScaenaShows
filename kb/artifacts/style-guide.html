ScaenaComposer — Design System
Pre-build reference · Visual language, component patterns, and token definitions · ScaenaShows v2
1 · Design Philosophy

ScaenaComposer is backstage, not onstage. It is a production tool — dense, precise, and dark. The theatrical identity is present structurally (in vocabulary and naming) rather than decoratively. Gold is used as the single brand accent; it connects the tool to ScaenaCraft without performing it.

Dark by default. The tool is used at a desk, often in a dimmed or creative environment. A dark background reduces eye strain during long composition sessions and makes colored event bars read with maximum clarity.

Color encodes category. Timeline lane colors are the primary information layer for event type at a glance. All other color usage is subordinate to this. Don't repurpose a lane color for any other meaning.

Density is a feature. The tool needs to show 13 lanes, a spatial canvas, properties, a library, and a chat interface simultaneously. Compact rows (24px lane height), 9–10px secondary labels, and minimal padding are correct here. This is a pro tool, not an onboarding flow.

Purple is ShowSprite's color. The purple family (#C080FF, deep purple backgrounds) belongs to ShowSprite. It is not used for any other component. When purple appears, it means ShowSprite.

7 · Firework Tile Visual Language

Firework tiles are rendered on the 2D spatial canvas. Each tile communicates: shape, primary colors (proportional), trail, flicker, and chase sequence number.

Shapes

Color Variation Modes (FIREWORK_CIRCLE)

UNIFORM

All positions use preset colors unchanged

RAINBOW

Cycles through fixed 7-color array per position

GRADIENT

Interpolates between gradient_from → gradient_to

ALTERNATE

Flips between preset's first and second star colors

Multi-Color Star — Proportional Segment Rendering

When a star has multiple primary colors, the tile circle is divided into proportional arc segments. Color list ["#FFF","#FFF","#FFF","#FFD700"] → ¾ white, ¼ gold. The ratio communicates relative particle density in the actual firework.

Left: 50/50 (#FECB00, #FFF) — scaena_gold preset
Center: 75/25 (#FFF×3, #FFD700×1) — texture_white_gold
Right: 33/33/33 (red, gold, white) — three-color example
8 · ShowSprite — Diff / Approve Card

When ShowSprite proposes a change, it appears as a structured diff card above the chat input. Each card can be approved or rejected independently. No change is applied without approval.

insert_event Insert FIREWORK_CIRCLE at tick 80 in ramp.warm_gold.01
- at: 0
type: SOUND
sound_id: minecraft:block.note_block.chime
- at: 0
type: FIREWORK_CIRCLE
...
+ - at: 80
+ type: FIREWORK_CIRCLE
+ preset: scaena_gold
+ radius: 3
+ count: 4
+ y_mode: surface
+ y_offset: 4
+ chase:
+ enabled: true
+ direction: FL
+ interval_ticks: 4
This will add 1 event to ramp.warm_gold.01 · Unsaved until you Save

Cards stack above the chat when multiple proposals are pending. Each is independently approvable. Approved changes are applied to the Cue in memory immediately; the file is not written until the creator saves.

9 · Open Design Questions — Pre-Build

These are visual/interaction design questions that need decisions before implementation begins. They do not affect the YAML schema or plugin — only the composer UI.

#QuestionOptionsRecommended
UI-1 How does the Cue library handle a large number of cues? (50+) Paginate · Virtualize list · Group by category letter Virtualize (react-virtual or similar)
UI-2 How does inline Cue expansion behave on the timeline? (hybrid nav) Expand in-place pushing lanes down · Replace current view · Open in split pane Expand in-place for 1 level; replace+breadcrumb for deeper
UI-3 Error display for validation (missing cue ID, cycle detected, etc.) Inline red bar on offending cue · Toast notification · Properties panel error state Inline red bar + properties panel error, no toast
UI-4 Unsaved changes indicator Dot on Save button · Modified filename · Full banner Dot on Save button (like VS Code) + title bar asterisk
UI-5 Keyboard shortcuts spec — which actions need them? Minimal (Save, Play, Stop) · Full DAW-style (nudge ticks, etc.) Start minimal; add as patterns emerge during use
